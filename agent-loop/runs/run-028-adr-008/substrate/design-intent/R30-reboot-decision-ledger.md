## R30 — Derive/promote authority seam (Operational staging tier vs. EA-gated authoritative tier)
- **Ruling:** Distillation *deriving* an `ObservedPattern` into the Operational plane is a permitted non-EA-gated `derived` write (distilled track-record, `derivation_class: distilled`, never authoritative selection knowledge); *promoting* that pattern into authoritative selection knowledge (Catalog / Standards) remains EA-gated via the feedback loop. The Operational plane is the **staging tier**; the EA gate sits between it and the **authoritative tier**. This is why a non-EA-gated KG write does not breach ADR-001's SOFIA-never-self-modifies invariant. Ratified 2026-06-22 in the RBT-14 (DDR-003) design session.
- **Rationale:** DDR-002 §2.3 already writes `ObservedPattern` as `origin_mechanism: derived` and routes weakness/strength *entry into authoritative knowledge* through the EA gate — but the apparent self-modification tension (a KG write with no EA gate vs. "SOFIA never self-modifies the KG") was never made explicit. The seam resolves it: the self-modification invariant governs *authoritative ground truth*, not a `derived` distillation into a staging tier. Cross-cutting because it reconciles two ratified ADR/DDR commitments rather than living inside DDR-003's policy.
- **Options declined:** **Gate all KG writes** — forces the distillation step through the EA gate, which neither DDR-001 nor DDR-002 intends and which makes the Operational plane unwritable without per-pattern EA action. **Leave the seam implicit** — ships an `ObservedPattern` write path that reads as self-modification and invites an antagonistic-review flag.
