## 5. Coalesced docket — 7 decisions (ratification per item; groups splittable)

**D1 — §2.2's absolute mechanism clause exceeds its ratified basis.** *(3: `d6114d1d` LAA-p1, `972b81cc` EA-p2, `c35f527a` coh-p2.)* The clause "no architectural mechanism may exist whose function depends on preventing it" (+ §5.2 restatement) is broader than the ratified "never precluded" and rides in prose without its own ratification trail. **Recommendation: ratify the clause explicitly as a deliberate commitment** (it is the teeth that stop "we didn't preclude it, we just made it impossible"), recorded in the deliberation record as a docket ruling — converting smuggled to ratified without text change; alternatively narrow to the ratified sentence and move the mechanism language into §6 check 2 as flag criteria.

**D2 — §2.3's "no routing choice can bypass a gate" is asserted, not derived.** *(5: `97991fb7` SA-p1, `21817b3c` SA-p2, `e77a9fba` coh-p2, `7b4e7495` LAA-p3, `bc37ba82` EA-p3 — all four hats.)* A direct interaction performing no graph write reaches no gateway and no graph-anchored checkpoint; the unbypassability claim supplies no mechanism. The answer exists upstream and the record failed to state it: gates anchor to **authoritative-state transitions**, and ADR-002 §2.1/§2.5 make the sole-owner gateway the only path to authoritative state — so a gate binds when its checkpoint's state transition is attempted, on any route, and an interaction that never touches authoritative state passes no gate because nothing of record occurred; its reasoning-bearing outcome still enters through capture. **Recommendation: state X — derive unbypassability from the sole-path property (cite ADR-002 §2.1/§2.5); qualify Y — rephrase "no routing choice can bypass a gate" to "no routing choice can carry a gated state transition past its gate, because every authoritative transition executes at the gateway."**

**D3 — predicate-routing attribution (body already author-fixed; ratify + finish).** *(4 DB pre-edit: `ab975421` LAA-p1, `db01b157` EA-p1, `8d9bcfa1` coh-p1, `dc103ab0` coh-p2; + 3 open resolvables post-edit: `44ba436b`, `1deab3f9`, `cb3cdef2`.)* v0.1.0 attributed the predicate-semantics routing to "DDR-003's routing"; DDR-003 routes only the Condition predicate *vocabulary*, while the single-source designation is SDD-001 §4.2's. The run author's pass-2 body edit already re-grounded §2.4 correctly (DDR-002 §3 obligation-closure + SDD-001 §4.2 single-source) and fixed §7's DDR-003 line to "Condition predicate vocabulary"; pass-3 resolvables note residual body/§7 phrasing lag. **Recommendation: ratify the author's re-grounding and complete the §7 alignment in the same amendment.**

**D4 — the model-access-gateway instance violates §2.6's own canon-named rule.** *(8: `7541a348` LAA-p1, `49bad287` SA-p1, `c750b13c` EA-p1, `617d622c` LAA-p2, `5854508b` SA-p2, `2cf2c339` coh-p2, `d86ceb0a` EA-p2, `103f66d5` EA-p3 — all four hats, three passes.)* No live canon names an ai-gateway/model-access gateway (SDD-001 §2.2 explicitly declines to); §2.6 names it as an instance inside a rule that permits canon-named components only. **Recommendation: drop the instance from §2.6 and §5.1** (knowledge-service alone anchors the substrate-service class; §5.1's unblocked list keeps the two canon-named agent designs and refers generically otherwise); ai-gateway-service remains in §7 as a forthcoming design, its existence decided by its own SDD — which is the roster-as-consequence discipline applied to ourselves.

**D5 — the recursion clause commits what §2.7 defers (ReasoningSession ownership).** *(9: `7ff5add0` EA-p1, `0965653f` LAA-p2, `210ac6cb` coh-p2, `6b5b38b1` SA-p2, `a30820bf` EA-p2, `5863b4a7` SA-p3, `e04d4dae` coh-p3, `2e545ae9` LAA-p3, `850868c4` EA-p3 — the run's largest cluster, all four hats.)* "A sub-run is a run" + §2.5's per-run `ReasoningSession` + composing-agent-as-orchestrator jointly require either non-AOE session ownership (contradicting ADR-002 §2.6) or a nesting shape DDR-002 §4 doesn't carry — the exact widening §2.7 says needs an upstream amendment first.
